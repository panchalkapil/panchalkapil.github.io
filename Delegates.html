Delegates in C#.

Delegates in C# are similar like Pointers in C/C++, and these are type safe, secure when compare to Pointers. Whoever faced this word 'Delegates' at beginning could have felt confused but believe me this is very simple to understand. In simple words, delegate holds the reference of several methods and invokes/calls them whenever needed.

Let me put you with real world example to understand it better.

For Example you are in a Bus station, and suddenly you hear fire alarm how would you react to it? You will run immediately out of the station, right. Now lets put this scenario in delegates.

The fire alarm you heard is nothing but an event. Once you heard that alarm you yourself initiated a run() function to escape from there.

But when you hear an announcement about your Bus you reserved, you don't escape but move to that platform to catch the Bus. Similarly, if you hear an announcement of your name for someone's phonecall, you will go straight a way to inquiry counter to attend the call.

Therefore, in real-world we also react differently based on the situation or event. In Programming, delegates does the same thing. We create number of appropriate functions and execute them based on particular events.

Delegates in Programming :

Basically a delegate consist of 3 steps:

  • Declaration
  • Instantiation
  • Invocation

Now Open your visual studio and create a new console application and follow the coding shown below:

using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;
 
namespace ConsoleApplication2
{
    public class DelegateExample
    {
        //Declaration
        //Creating Delegates with no parameters and no return type.
        public delegate void FirstDelegate();
 
        public void func1()
        {
            Console.WriteLine("I am Function 1");
        }
        public void func2()
        {
            Console.WriteLine("I am Function 2");
        }
        public void func3()
        {
            Console.WriteLine("I am Function 3");
        }
    }
 
    class Program
    {
        static void Main(string[] args)
        {
      DelegateExample testdelegate = new DelegateExample();
//Instantiation
      DelegateExample.FirstDelegate fd1 = new DelegateExample.FirstDelegate(testdelegate.func1);
			
      DelegateExample.FirstDelegate fd2 = new DelegateExample.FirstDelegate(testdelegate.func2);
			
      DelegateExample.FirstDelegate fd3 = new DelegateExample.FirstDelegate(testdelegate.func3);
 
//Invocation 
           fd1();
            fd2();
            fd3();
 
            Console.ReadKey();
        }
    }
}

The output for the above program of delegates will look like this below.


OUTPUT:

I am Function 1

I am Function 2

I am Function 3

What we did in the above Example Program?

  • In the above example we have created a delegate without passing parameters.
    • public delegate void FirstDelegate();
  • Then after we have created 3 functions.
    • public void func1()
    public void func2()
    public void func3()
  • And then inside the main function, we have created an object of delegate. One important thing here to remember about delegate is, it always require a function during initialization. 
    • DelegateExample.FirstDelegate fd1 = new DelegateExample.FirstDelegate(testdelegate.func1);

    DelegateExample.FirstDelegate fd2 = new DelegateExample.FirstDelegate(testdelegate.func2);
    DelegateExample.FirstDelegate fd3 = new DelegateExample.FirstDelegate(testdelegate.func3);

And at the end we have called the delegates to execute the functions.

fd1();
fd2();
fd3();

Some important points to remember about Delegates:
  • We can easily call multiple methods on a single event using delegates.
  • It is not necessary for the methods to match the exact signature of the delegate.
  • Delegates allows us to pass methods as parameter.
  • It is used for asynchronous programming.
  • It is useful when there is a need of creating anonymous function.
  • These are also used to define callback methods.
